# Kubernetes
# Container Orchestration
 - Manage Availability, Scaling and networking of Container
 - Monitoring the cluster (group of host)
 - Manage the timing of container creation
 - Help in container configuration in order to allow container to communicate with one another

# Features
 - Automatic bin-packaging
 - Horizontal Scaling
 - Self healing
 - Batch execution
 - Storage Orchestration

 # Kubernetes Cluster Architecture
  - Master Node: Manages Kubernetes API Server, resource Controller and Scheduler
  - Worker Node: Minions (Physical or Virtual Machine)
  - Pod: Group of Container tightly coupled together
